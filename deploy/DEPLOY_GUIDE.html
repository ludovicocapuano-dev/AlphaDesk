Configurare l'accesso SSH al VPS Hetzner
Genera una chiave SSH sul tuo Mac/PC
Apri il terminale sul tuo computer locale (non il VPS) e genera una coppia di chiavi SSH.
# Sul tuo computer locale (Mac/Linux) ssh-keygen -t ed25519 -C "ludovicocapuano@gmail.com"
Premi Enter per accettare il percorso default (~/.ssh/id_ed25519). Inserisci una passphrase per sicurezza extra, oppure premi Enter per saltarla.
Aggiungi la chiave SSH al progetto Hetzner
Copia la chiave pubblica e aggiungila su Hetzner Cloud.
# Copia la chiave pubblica cat ~/.ssh/id_ed25519.pub
Vai su cloud.hetzner.com → Security → SSH Keys → Add SSH Key. Incolla il contenuto e dai un nome alla chiave.
Crea il server (se non l'hai già fatto)
Su Hetzner Cloud → Servers → Create Server.
Configurazione consigliata per AlphaDesk:
• Location: Falkenstein o Helsinki (bassa latenza verso mercati EU)
• Image: Ubuntu 24.04
• Type: CX22 (2 vCPU, 4GB RAM) — abbondante per il bot, ~€4/mese
• SSH Key: seleziona quella appena aggiunta
• Networking: IPv4 (serve per API esterne)
Collegati via SSH
Dopo la creazione, Hetzner ti mostra l'IP del server. Collegati:
# Sostituisci con il tuo IP ssh root@TUO_IP_SERVER
Se è la prima volta, conferma con yes quando chiede la fingerprint.
Crea un utente dedicato (non usare root)
Per sicurezza, crea un utente separato per il bot:
# Ancora connesso come root adduser alphadesk usermod -aG sudo alphadesk # Copia la chiave SSH per il nuovo utente mkdir -p /home/alphadesk/.ssh cp ~/.ssh/authorized_keys /home/alphadesk/.ssh/ chown -R alphadesk:alphadesk /home/alphadesk/.ssh chmod 700 /home/alphadesk/.ssh chmod 600 /home/alphadesk/.ssh/authorized_keys
Da ora in poi, connettiti sempre come ssh alphadesk@TUO_IP, non come root.
Configura il firewall base
# Connesso come alphadesk sudo ufw allow OpenSSH sudo ufw enable sudo ufw status
2
Installare l'ambiente e le dipendenze
Aggiorna il sistema e installa i prerequisiti
sudo apt update && sudo apt upgrade -y sudo apt install -y python3 python3-pip python3-venv git curl unzip htop
Installa Node.js 22 (serve per Claude Code)
# Installa Node.js 22 LTS via NodeSource curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - sudo apt install -y nodejs # Verifica node --version # deve essere v22.x.x npm --version
Carica il progetto AlphaDesk sul VPS
Dal tuo computer locale, carica la cartella alphadesk:
# Dal tuo computer locale (NON dal VPS) scp -r ./alphadesk alphadesk@TUO_IP:~/alphadesk
Alternativa: puoi mettere il progetto su un repo Git privato e fare git clone dal VPS.
Crea l'ambiente Python e installa le dipendenze
# Sul VPS, connesso come alphadesk cd ~/alphadesk python3 -m venv venv source venv/bin/activate pip install --upgrade pip pip install -r requirements.txt # Verifica che funzioni python -c "import httpx, numpy, pandas, apscheduler; print('Tutte le dipendenze OK')"
Configura le credenziali (.env)
# Crea il file .env cd ~/alphadesk nano .env
Incolla questo contenuto, sostituendo i valori:
# eToro API ETORO_USER_KEY=la_tua_user_key_qui ETORO_API_KEY=la_tua_api_key_qui ETORO_ENV=Demo # Telegram (opzionale ma consigliato) TELEGRAM_BOT_TOKEN=il_tuo_bot_token TELEGRAM_CHAT_ID=il_tuo_chat_id
Salva con Ctrl+O → Enter → Ctrl+X
Non condividere mai il file .env. Proteggi i permessi:
chmod 600 .env
Test: esegui il bot una volta manualmente
cd ~/alphadesk source venv/bin/activate set -a && source .env && set +a # Test singolo ciclo python main.py
Se vedi "AlphaDesk initializing..." e non ci sono errori fatali, il sistema funziona.
3
Installare Claude Code sul VPS
Installa Claude Code
Usa l'installer nativo di Anthropic (raccomandato, non serve npm):
# Installer nativo (metodo raccomandato) curl -fsSL https://claude.ai/install.sh | bash # Verifica claude --version
Se l'installer nativo non funziona, usa il metodo npm come fallback: npm install -g @anthropic-ai/claude-code
Autenticazione Claude Code
Dato che il VPS non ha browser, usa l'autenticazione via API key:
# Opzione A: API Key Anthropic (consigliata per VPS) export ANTHROPIC_API_KEY=la_tua_api_key_anthropic echo 'export ANTHROPIC_API_KEY=la_tua_api_key_anthropic' >> ~/.bashrc # Avvia Claude Code nella cartella del progetto cd ~/alphadesk claude
Per ottenere l'API key Anthropic, vai su console.anthropic.com → API Keys. Claude Code la usa per funzionare senza browser.
Alternativa — login headless (se hai un account Claude Pro/Team):
# Opzione B: OAuth headless claude login # Ti darà un URL da aprire nel browser del tuo PC # Apri l'URL, autorizza, copia il codice che appare # e incollalo nel terminale del VPS
Test: usa Claude Code sul progetto
cd ~/alphadesk claude # Dentro Claude Code, prova: # "Analizza la struttura del progetto e dimmi se ci sono errori" # "Esegui il backtest con dati yfinance reali" # "Mostrami le posizioni aperte sull'account eToro Demo"
Ora hai Claude Code che può leggere, modificare ed eseguire il tuo bot direttamente dal VPS.
4
Deploy H24 con systemd
Crea il servizio systemd
sudo nano /etc/systemd/system/alphadesk.service
Incolla questo contenuto:
[Unit] Description=AlphaDesk Quant Trading System After=network.target StartLimitIntervalSec=300 StartLimitBurst=5 [Service] Type=simple User=alphadesk WorkingDirectory=/home/alphadesk/alphadesk ExecStart=/home/alphadesk/alphadesk/venv/bin/python scheduler.py EnvironmentFile=/home/alphadesk/alphadesk/.env Restart=always RestartSec=30 StandardOutput=journal StandardError=journal SyslogIdentifier=alphadesk [Install] WantedBy=multi-user.target
Attiva e avvia il servizio
# Ricarica systemd sudo systemctl daemon-reload # Abilita avvio automatico al boot del server sudo systemctl enable alphadesk # Avvia il bot sudo systemctl start alphadesk # Controlla che sia attivo sudo systemctl status alphadesk
Se vedi Active: active (running) in verde, il bot gira H24!
Impara i comandi di gestione del servizio
# Comandi che userai quotidianamente: # Vedere i log in tempo reale sudo journalctl -u alphadesk -f # Vedere gli ultimi 100 log sudo journalctl -u alphadesk -n 100 # Riavviare il bot (dopo modifiche al codice) sudo systemctl restart alphadesk # Fermare il bot sudo systemctl stop alphadesk # Stato del bot sudo systemctl status alphadesk # Vedere i log applicativi tail -f ~/alphadesk/logs/alphadesk.log

Monitoring e manutenzione
Configura il Telegram Bot per le notifiche
Segui la guida in deploy/TELEGRAM_SETUP.md nel progetto. Una volta configurato, testa:
cd ~/alphadesk && source venv/bin/activate set -a && source .env && set +a python -c " import asyncio from utils.telegram_bot import TelegramNotifier import os async def test(): bot = TelegramNotifier(os.getenv('TELEGRAM_BOT_TOKEN'), os.getenv('TELEGRAM_CHAT_ID')) await bot.send('🟢 AlphaDesk su VPS Hetzner — Connessione OK!') print('Messaggio inviato!') asyncio.run(test()) "
Workflow quotidiano con Claude Code
Ecco come usare Claude Code per gestire il bot giorno per giorno:
# Collegati al VPS ssh alphadesk@TUO_IP # Apri Claude Code nel progetto cd ~/alphadesk && claude # Ora puoi chiedere a Claude cose come: # "Controlla i log delle ultime 2 ore, ci sono errori?" # "Mostrami il report di performance dell'ultima settimana" # "Il momentum strategy ha win rate basso, prova ad alzare # il volume_threshold a 2.0 e riesegui il backtest" # "Aggiungi il ticker PLTR all'universo equity" # "Esegui un backtest con i dati reali e genera il report" # "Riavvia il servizio alphadesk"
Questo è il vero power-up: Claude Code sul VPS ti permette di iterare, debuggare e migliorare il bot in linguaggio naturale, senza dover scrivere codice manualmente ogni volta.
